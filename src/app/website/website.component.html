Website

This website is made with a plan. There is an approach wich defines an order that should be followed. There are requirements to define when the website is complete, and to ensure integrety of the website in the future.

Requirements

Number Definition Completed
1 The website must be available. ✓
1.1 The website must be available on mobile web browsers. ✓
1.1.1 The website should be responsive. ✓
1.2 The website must be available on desktop web browsers. ✓
1.3 The website must be available on web browsers without javascript. ✓
2 The website must have an one-page design. ✓
3 The website should be accesible.
3.1 The website could support multiple languages.
3.2 The website should support text-to-speech aids.
4 The website must have editor notes for further explanation. ✓
4.1 The website should have an option to collapse editor notes.
5 The website content must be complete.
5.1 The website introduction content must be filled. ✓
5.2 The website about content must be filled. ✓
5.3 The website team content must be complete.
5.3.1 The website team profiles should have a respective picture.
5.3.2 The website team profiles must have a default picture.
5.3.3 The website team profiles should have a respective description.
5.4 The website skills content must be complete.
5.5 The website passions content must be filled. ✓
5.6 The website meta 'website' content must be complete. ✓
6 The website changes must be tracked by a git repository. ✓
6.1 The website repository should be hosted on Github, as backup.
6.2 The website repository must not be responsible for tracking sensitive personal information.
7 The website uses Angular for seperation of concerns.

Approach

The first version of the website is made in the following order.

  1. Have an idea.
  2. Define requirements through MoSCoW.
  3. Setup local web server.
  4. Add git repository hosted on Github.
  5. Add HTML.
  6. Add CSS through Sass.
    1. Add taskrunner.
  7. Review
    1. Review by others, increases quality.
